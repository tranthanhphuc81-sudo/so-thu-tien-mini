📌 Tổng quan hệ thống

Sổ Thu Tiền Mini là công cụ quản lý thu tiền theo nhóm, dành cho đội ngũ nhờ cộng tác viên (CTV) trực tiếp nhận tiền từ khách hàng. Dữ liệu đồng bộ real-time qua Firebase — NVTD và toàn bộ CTV luôn thấy trạng thái mới nhất.

👤 NVTD (Người vận hành)

  • Nhập danh sách khách hàng
  • Phân nhóm khách hàng cho CTV
  • Cấp / thu hồi quyền đăng nhập CTV
  • Reset mật khẩu CTV nếu cần
  • Xem tổng quan, tiến độ & cảnh báo số tiền
  • Kiểm tra bill do CTV xác nhận
  • Tạo QR thanh toán & rút gọn link CTV

👥 CTV (Cộng tác viên)

  • Xem danh sách KH được phân công
  • Xác nhận đã thu tiền (có bill & cảnh báo)
  • Tạo QR để KH chuyển khoản
  • Chia sẻ QR qua Zalo
  • Tra cứu KH đã / chưa thu

🔥 Bước 1: Tạo dự án Firebase Bắt buộc
Bước này chỉ cần làm một lần duy nhất cho mỗi NVTD. Sau khi kết nối xong, dữ liệu tự đồng bộ với mọi thiết bị.
  1. Truy cập Firebase Console
    Mở trình duyệt, vào console.firebase.google.com → Đăng nhập bằng tài khoản Google.
  2. Tạo dự án mới
    Bấm Add project → Đặt tên dự án (ví dụ: SoThuTien-HoaLan) → Tắt Google Analytics → Bấm Create project.
  3. Tạo Realtime Database
    Trong menu trái → Build → Realtime Database → Bấm Create Database → Chọn vị trí asia-southeast1 (Singapore) → Chọn Start in test mode → Bấm Enable.
  4. Lấy Database URL
    Trang RTDB sẽ hiện URL dạng:
    https://ten-du-an-default-rtdb.asia-southeast1.firebasedatabase.app
    Copy URL này lại.
  5. Lấy API Key
    Menu trái → biểu tượng bánh răng ⚙️ → Project settings → Kéo xuống mục Your apps → Bấm icon </> (Web) → Đặt tên app → Bấm Register app → Copy giá trị apiKey từ đoạn config hiện ra.
  6. Nhập vào ứng dụng
    Mở nvtd.html → Tab ⚙️ Thiết lập → Dán API Key và Database URL vào ô tương ứng → Bấm 🔥 Kết nối Firebase.
Sau khi kết nối thành công, badge trạng thái sẽ đổi thành 🟢 Đã kết nối Firebase.
🛡️ Bước 2: Cập nhật Firebase Rules Quan trọng
Nếu bỏ qua bước này, mọi người đều đọc/ghi được dữ liệu của bạn. Phải làm ngay sau khi tạo database.
  1. Vào trang Rules
    Firebase Console → Realtime Database → Bấm tab Rules.
  2. Copy nội dung Rules mới
    Quay lại nvtd.html → Tab ⚙️ Thiết lập → Kéo xuống thẻ 🛡️ Bảo mật Firebase → Bấm 📋 Copy Rules JSON.
  3. Dán và Publish
    Xóa hết nội dung cũ trong ô Rules trên Firebase Console → Dán nội dung vừa copy → Bấm Publish.

Nội dung Rules cần dán:

{
  "rules": {
    ".read": false,
    ".write": false,
    "ctvAllowed": {
      ".read": true,
      ".write": "auth != null && auth.token.email === 'nvtd@sothu.app'"
    },
    "ctvForceReset": {
      ".read": true,
      ".write": "auth != null"
    },
    "data": {
      ".read": "auth != null",
      ".write": "auth != null"
    }
  }
}

🔑 Bước 3: Bật Email/Password Authentication Quan trọng
  1. Vào mục Authentication
    Firebase Console → Menu trái → Build → Authentication.
  2. Bật phương thức đăng nhập
    Bấm tab Sign-in method → Tìm dòng Email/Password → Bấm vào → Bật toggle Enable → Bấm Save.
Sau khi bật xong, quay lại nvtd.html và tạo mật khẩu NVTD lần đầu (xem phần 🔐 Tạo mật khẩu NVTD).
🏦 Bước 4: Nhập thông tin ngân hàng

Thông tin này được dùng để tạo mã QR thanh toán cho từng khách hàng.

  1. Mở thẻ Thông tin tài khoản ngân hàng
    Tab ⚙️ Thiết lập → Kéo đến thẻ 🏦 Thông tin tài khoản ngân hàng.
  2. Điền đầy đủ
    • Ngân hàng: Chọn từ danh sách (MB, VCB, TCB, BIDV, ACB...)
    • Số tài khoản: Nhập đúng số TK nhận tiền
    • Tên chủ tài khoản: Nhập IN HOA không dấu (ví dụ: NGUYEN VAN AN)
  3. Lưu
    Bấm 💾 Lưu thông tin. Hệ thống sẽ đồng bộ lên Firebase ngay.
QR thanh toán được tạo tự động từ thông tin này theo chuẩn VietQR. CTV cũng dùng thông tin này để tạo QR khi cần.
📥 Bước 5: Nhập danh sách khách hàng

Cách 1: Dán trực tiếp (nhanh nhất)

  1. Nhập tên CTV phụ trách
    Ô Tên CTV phụ trách → nhập tên của CTV sẽ đi thu nhóm này (ví dụ: Hoa, Lan).
  2. Dán danh sách
    Dán danh sách vào ô lớn, mỗi dòng một khách, định dạng:
    Tên Khách Hàng | Số tiền | Mã đối soát

    Ví dụ: 
    Nguyễn Văn An | 500000 | KH001
Trần Thị Bình | 800000 | KH002
Lê Văn Cường | 1200000
    💡 Nếu không có mã đối soát thì có thể bỏ phần | KH...
  3. Nhập
    Bấm 📥 Nhập danh sách. Dữ liệu được thêm vào (không xóa dữ liệu cũ).
Muốn xóa toàn bộ dữ liệu và nhập lại từ đầu → Bấm 🗑️ Xóa tất cả dữ liệu trước. Thao tác này không thể hoàn tác.
📊 Nhập từ file Excel Tiện lợi

Hỗ trợ file .xlsx và .xls. Hệ thống tự động nhận dạng cột tên khách hàng, mã KH và số tiền.

  1. Chọn CTV phụ trách
    Nhập hoặc chọn tên CTV trong ô CTV phụ trách danh sách này.
  2. Upload file
    Kéo thả file Excel vào vùng tải lên, hoặc bấm vào để chọn file từ máy.
  3. Kiểm tra cột
    Hệ thống hiển thị bảng xem trước và gợi ý cột. Chỉnh lại nếu cần: chọn đúng cột Tên KH, Số tiền, Mã đối soát.
  4. Xác nhận nhập
    Bấm ✅ Nhập N khách hàng.
🌐 Bước 6: Đưa ứng dụng lên web Bắt buộc để gửi link CTV
Khi chạy file HTML trực tiếp từ máy (file://...), link tạo ra không gửi được cho CTV. Cần đưa lên một địa chỉ web thật.

Cách dễ nhất: Netlify Drop (miễn phí, không cần tài khoản)

  1. Mở Netlify Drop
    Vào app.netlify.com/drop.
  2. Kéo thả thư mục
    Kéo thả toàn bộ thư mục SoThuTienMini (chứa 3 file HTML) vào khung Netlify. Không kéo từng file, phải kéo cả thư mục.
  3. Lấy địa chỉ
    Netlify sẽ cấp địa chỉ kiểu https://abc123.netlify.app. Copy địa chỉ này.
  4. Lưu vào ứng dụng
    Mở nvtd.html → Tab ⚙️ Thiết lập → Thẻ 🌐 Địa chỉ website → Dán URL → Bấm 💾 Lưu URL.
Tùy chọn khác: GitHub Pages (miễn phí, cần tài khoản GitHub), Vercel, hoặc bất kỳ hosting nào có HTTPS. Chỉ cần đưa 3 file .html lên là đủ.
Mỗi khi cập nhật file HTML, cần kéo thả lại lên Netlify để CTV nhận phiên bản mới. URL sẽ giữ nguyên nếu cùng site.
🔐 Tạo mật khẩu NVTD lần đầu

NVTD đăng nhập bằng Firebase Email/Password với email cố định nvtd@sothu.app. Mật khẩu do bạn tự đặt và lưu bên phía Firebase — không ai khác biết.

  1. Kết nối Firebase thành công
    Phải hoàn thành Bước 1–3 trước.
  2. Màn hình đăng nhập hiện ra
    Sau khi kết nối Firebase, giao diện đăng nhập xuất hiện với 2 lựa chọn:
    Chưa có mật khẩu → Bấm Tạo mật khẩu lần đầu
    Đã có mật khẩu → Dùng form Đăng nhập bình thường
  3. Đặt mật khẩu
    Nhập mật khẩu mới (tối thiểu 6 ký tự) → Nhập lại xác nhận → Bấm ✅ Tạo mật khẩu.

Đổi mật khẩu về sau

Tab ⚙️ Thiết lập → Thẻ 🔐 Đổi mật khẩu NVTD → Nhập mật khẩu cũ → Nhập mật khẩu mới → Bấm 💾 Đổi mật khẩu.

⚠️ Nếu quên mật khẩu và không vào được app — bấm nút 🔑 Quên mật khẩu? ngay dưới ô đăng nhập; hoặc xem hướng dẫn chi tiết ở mục ➡ Quên mật khẩu NVTD bên dưới.
🔑 Quên mật khẩu NVTD

Vì email nvtd@sothu.app là email nội bộ (không nhận được thư), Firebase không thể gửi email khôi phục thông thường. Thay vào đó, bạn xóa tài khoản cũ và tạo lại — dữ liệu không bị mất.

Cách 1: Dùng nút trong app (nhanh nhất)

  1. Mở trang đăng nhập NVTD
    Từ trình duyệt, mở lại nvtd.html. Khi màn hình đăng nhập hiện ra, bấm nút 🔑 Quên mật khẩu? (ngay dưới ô mật khẩu).
  2. Mở Firebase Console
    Bấm 🌐 Mở Firebase Console → Authentication — nút sẽ mở thẳng trang danh sách user của dự án của bạn.
  3. Xóa tài khoản cũ
    Tìm dòng nvtd@sothu.app → bấm dấu ⋮ → Delete account → Xác nhận.
  4. Tạo lại mật khẩu
    Quay lại app → bấm 📝 Tạo mật khẩu mới ngay trong panel đó → Nhập mật khẩu mới → Xong!

Cách 2: Thủ công qua Firebase Console

  1. Vào Firebase Console
    Truy cập: console.firebase.google.com → Chọn dự án của bạn.
  2. Authentication → Users
    Menu trái → Build → Authentication → Tab Users.
  3. Xóa tài khoản
    Tìm nvtd@sothu.app → bấm ⋮ → Delete account. Dữ liệu khách hàng trong Realtime Database không bị ảnh hưởng.
  4. Tạo mật khẩu mới
    Mở lại nvtd.html → Tạo tài khoản → Đặt mật khẩu mới → Vào app bình thường.
Dữ liệu khách hàng, lịch sử thu tiền, thông tin ngân hàng văn đờu ở Firebase Realtime Database và không liên quan đến tài khoản Authentication. Xóa account chỉ xóa quyền đăng nhập, không xóa dữ liệu.

🚦 Cấp quyền đăng nhập cho CTV
CTV chỉ có thể tạo mật khẩu và đăng nhập sau khi NVTD cấp quyền. Chưa cấp quyền thì màn hình CTV hiện thông báo chờ.
  1. Mở tab Quản lý CTV
    Bấm tab 👥 Quản lý CTV trong nvtd.html.
  2. Tìm CTV cần cấp quyền
    Thẻ 🔑 Cấp quyền đăng nhập CTV hiển thị danh sách tất cả CTV. Dùng ô Tìm tên CTV... để lọc nhanh nếu có nhiều CTV.
  3. Cấp quyền
    Một CTV: Bấm ✅ Cấp quyền ngay trên dòng của CTV đó.
    Nhiều CTV cùng lúc: Tick checkbox → Bấm ✅ Cấp đã chọn hoặc ✅ Cấp tất cả.
  4. Gửi link cho CTV
    Tab ⚙️ Thiết lập → Thẻ 🔗 Link công cụ CTV → Mỗi CTV có link riêng → Bấm 📋 Copy hoặc dùng nút ✂️ Rút gọn để lấy link ngắn trước khi gửi. Xem chi tiết tại mục Link CTV & Rút gọn.

Thu hồi quyền

Tick checkbox CTV → Bấm 🚫 Thu hồi đã chọn. CTV đó sẽ không đăng nhập được nữa. Dữ liệu họ đã xác nhận vẫn giữ nguyên.

🔄 Reset mật khẩu CTV

NVTD có thể buộc một CTV phải tạo mật khẩu mới vào lần đăng nhập tiếp theo — hữu ích khi CTV quên mật khẩu hoặc cần bảo mật lại.

Phía NVTD

  1. Mở thẻ Cấp quyền
    Tab 👥 Quản lý CTV → Thẻ 🔑 Cấp quyền đăng nhập CTV.
  2. Bấm Reset mật khẩu
    Tìm dòng CTV cần reset → Bấm nút 🔄 Reset mật khẩu. Nút đổi thành ⏳ Chờ đổi mật khẩu — flagged cho CTV biết.
  3. Hủy nếu cần
    Bấm lại nút đó → Chọn hủy. Flag được xóa và CTV không cần đổi mật khẩu nữa.

Phía CTV

Khi CTV đăng nhập trong khi có flag reset, sau khi xác thực xong sẽ xuất hiện màn hình bắt buộc đổi mật khẩu trước khi vào app. CTV nhập mật khẩu mới, nhập lại và xác nhận — sau đó vào app bình thường.

Dữ liệu đã xác nhận không bị mất. “Reset” chỉ buộc CTV đặt lại mật khẩu đăng nhập, không xóa bất kỳ dữ liệu nào.
🔗 Link công cụ CTV & Rút gọn

Trong tab ⚙️ Thiết lập → Thẻ 🔗 Link công cụ CTV, mỗi CTV có một link riêng để gửi.

NútTác dụng
📋 CopySao chép link đầy đủ vào clipboard
🔗 TestMở link trong tab mới để kiểm tra

Link chứa tham số cfg= (Firebase config dưới dạng base64) nên khá dài. CTV chỉ cần mở link một lần — app tự lưu vào bộ nhớ trình duyệt, lần sau không cần link dài nữa. Dùng ✂️ Rút gọn nếu muốn chia sẻ link gọn qua Zalo.
💳 Hướng dẫn dành cho CTV

CTV nhận link từ NVTD (dạng https://...netlify.app/ctv.html?ctv=TenCTV&cfg=...) và thực hiện theo các bước sau.

Lần đầu sử dụng

  1. Mở link
    Bấm vào link NVTD gửi. Trên điện thoại có thể lưu vào Cloud Zalo để tiện mở lại.
  2. Tạo mật khẩu lần đầu
    Màn hình đăng nhập hiện ra → Bấm link Lần đầu? Tạo mật khẩu → Nhập mật khẩu (ít nhất 6 ký tự) → Nhập lại → Bấm ✅ Tạo mật khẩu & Vào app.
  3. Nếu thấy thông báo "Chưa được cấp quyền"
    Liên hệ NVTD để được cấp quyền, sau đó mở lại link.

Xác nhận thu tiền

  1. Chọn khách hàng
    Tab ⏳ CHƯA THU → Tìm tên KH → Bấm ✅ Xác nhận đã thu.
  2. Tải ảnh bill (khuyến nghị)
    Bấm vùng tải ảnh → Chọn ảnh chụp từ điện thoại. QR, biên lai, hoặc ảnh chụp màn hình chuyển khoản đều được.
  3. Thêm ghi chú (tùy chọn)
    Ô ghi chú: ghi thêm thời gian, cách thức thu, hoặc mã giao dịch nếu cần.
  4. Xác nhận
    Bấm ✅ Xác nhận đã thu. KH chuyển sang tab ✅ ĐÃ THU ngay lập tức.
⚠️ Cảnh báo số tiền khi xác nhận:
  • Số tiền = 0 → không thể xác nhận
  • Số tiền gấp hơn 3 lần số nợ gốc → cảnh báo ngôn có thể nhập sai, cần xác nhận thêm
  • Chênh lệch hơn 20% so với số nợ gốc → cảnh báo để kiểm tra trước khi tiếp tục

Tạo QR để khách chuyển khoản

  1. Mở QR
    Trong card KH → Bấm 🔲 QR. Mã QR VietQR hiện ra đúng số tiền của KH đó.

🔲 Tạo mã QR thanh toán (NVTD)

NVTD có thể tạo QR cho từng KH để in ra hoặc gửi trực tiếp.

  1. Mở tab Danh sách KH
    Bấm tab 📋 Danh sách KH.
  2. Tạo QR từng KH
    Trên dòng của KH → Bấm nút 🔲 QR. Modal QR hiện ra kèm tên KH, số tiền, thông tin TK.
  3. Tạo tất cả QR cùng lúc
    Bấm 🖨️ Tạo tất cả QR → Tất cả QR được hiển thị để in (dùng Ctrl+P).
QR được tạo theo chuẩn VietQR — hỗ trợ hầu hết ứng dụng ngân hàng Việt Nam. Số tiền và mã đối soát được điền sẵn vào QR.
❓ Câu hỏi thường gặp
Nhiều NVTD có dùng chung app này được không?
Được. Mỗi NVTD tạo Firebase project riêng và nhập API Key của mình vào. Dữ liệu hoàn toàn độc lập — NVTD này không thấy dữ liệu của NVTD kia. Mọi người đều dùng cùng file HTML nhưng kết nối đến database khác nhau.
CTV có thể đăng nhập trên nhiều thiết bị không?
Được. CTV chỉ cần bấm link từ bất kỳ thiết bị nào → Nhập mật khẩu đã tạo → Vào app. Dữ liệu đồng bộ real-time trên mọi thiết bị.
NVTD quên mật khẩu và không vào được app
Trên màn hình đăng nhập, bấm 🔑 Quên mật khẩu? → bấm 🌐 Mở Firebase Console → Authentication → tìm nvtd@sothu.app → xóa account → quay lại app → tạo mật khẩu mới. Dữ liệu khách hàng không bị mất. Xem hướng dẫn chi tiết tại mục Quên mật khẩu NVTD.
CTV quên mật khẩu thì làm thế nào?
NVTD vào Firebase Console → Authentication → Users → Tìm email CTV (dạng ctv+...@sothu.app) → Xóa user đó đi → CTV mở lại link và tạo mật khẩu mới. Dữ liệu đã xác nhận trước đó không bị mất.
CTV thấy "Bạn chưa được cấp quyền" sau khi tạo mật khẩu
NVTD chưa cấp quyền cho CTV đó. Thực hiện: Tab 👥 Quản lý CTV → Thẻ 🔑 Cấp quyền → Chọn tên CTV → Bấm ✅ Cấp quyền đăng nhập. CTV mở lại link là vào được ngay.
Bị lỗi "Permission denied" khi đăng nhập xong
Firebase Rules chưa được cập nhật. Làm theo hướng dẫn ở Bước 2: Cập nhật Rules. App sẽ hiển thị nút 🔗 Mở Firebase Console → Rules dẫn thẳng tới trang cần sửa.
Lỗi "Email/Password Authentication chưa được bật"
Cần bật tính năng này trong Firebase Console. Xem hướng dẫn tại Bước 3: Bật Authentication.
Link gửi cho CTV quá dài, không gửi được qua Zalo
Lần đầu CTV mở link dài (chứa ?cfg=...), app tự lưu vào bộ nhớ trình duyệt. Các lần sau CTV chỉ cần bấm đúng link đó — không cần gửi lại link mới. Bạn cũng có thể dùng dịch vụ rút gọn link như zalo.me/s hoặc bit.ly.
Dữ liệu có bị mất không?
Dữ liệu lưu trên Firebase Realtime Database — không mất khi đóng trình duyệt, tắt máy, hay mở trên thiết bị khác. Firebase miễn phí có giới hạn 1 GB dung lượng và 10 GB/tháng băng thông — quá đủ cho quy mô vừa và nhỏ.
Làm sao đặt lại cấu hình Firebase (đổi project)?
Tab ⚙️ Thiết lập → Thẻ 🔥 Kết nối Firebase → Bấm 🔄 Reset & kết nối lại → Nhập API Key + DB URL mới → Kết nối.
Ứng dụng có dùng được offline không?
Giao diện vẫn hiển thị nhưng dữ liệu cần kết nối internet để đồng bộ. Khi mất mạng, Firebase sẽ cache dữ liệu cục bộ và tự đồng bộ lại khi có kết nối.
